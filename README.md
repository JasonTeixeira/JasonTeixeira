## Operating Principles

<table>
<tr>
<td width="33%" valign="top">

### Research Before Claims

Trading systems need clean data, realistic costs, walk-forward validation, regime awareness, and explicit anti-overfit checks before any result deserves attention.

</td>
<td width="33%" valign="top">

### Apps Before Slides

The work should become a usable product: admin dashboards, portals, APIs, billing, auth, observability, and deployment pipelines.

</td>
<td width="33%" valign="top">

### Cloud With Controls

Infrastructure should ship with least privilege, CI gates, audit logs, budget awareness, rollback paths, and security scanning from the first commit.

</td>
</tr>
